## 概要

以下 2 種類のチームオーナー権限管理 UI を実装する。
① **管理者向け**: ユーザー管理画面からグローバル `is_team_owner` フラグと `max_teams` を設定する
② **チームオーナー向け**: チーム詳細画面でメンバーのチーム内オーナーロールを付与・剥奪する（必須の注意ポップアップ付き）

---

## 背景・目的

TICKET-035 で追加した 2 層のチームオーナー権限を操作する管理 UI が必要。

---

## 受け入れ条件

### ① 管理者向け：グローバル team_owner 権限付与
- [x] ユーザー管理画面のユーザー詳細/編集で `is_team_owner` トグルを表示・操作できる（管理者のみ）
- [x] `is_team_owner: true` の場合、チーム作成上限数（`max_teams`）の数値入力フィールドを表示できる
- [x] 管理者はチーム詳細画面の「メンバーロール変更」操作も実行できる（下記②と同様のUIから）

### ② チームオーナー向け：チーム内オーナーロール付与・剥奪
- [x] チーム詳細画面のメンバー一覧に「オーナーにする」「オーナーを外す」ボタンを表示する（チームオーナー・管理者のみ）
- [x] 上記ボタンをクリックすると、実行前に**必ず**以下の内容を含む注意ポップアップを表示する
  - 対象のユーザー名を明示する
  - チームオーナー権限でできること（メンバー招待・退会・問題/タグの編集・削除・チーム内オーナーの付与）の再掲
  - 悪用リスクの警告（例：「不正なメンバーへの付与はチームデータの改ざんにつながります」）
  - 「付与する」「キャンセル」の 2 択ボタン（デフォルトはキャンセル）
- [x] チームにオーナーが自分 1 人の場合、「オーナーを外す」はできない（ボタン無効化 + 理由を tooltip で表示）

### チーム管理共通
- [x] チーム詳細画面でチームオーナーがメンバーを招待・退会させる UI を実装する
  - 招待: UUID/ユーザーID/eメールのいずれかで入力し送信
  - 退会: メンバー横のボタンで実行（確認ダイアログあり）
- [x] 配色・配置・コンポーネントは `global.css` の CSS カスタムプロパティと既存のボタン・フォームスタイルに従う
