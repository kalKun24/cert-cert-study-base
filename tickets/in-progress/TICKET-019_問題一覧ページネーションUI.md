## 概要

TICKET-008 でバックエンドのページネーション API（`page` / `per_page` / `total_pages`）が実装済みだが、フロントエンドの問題一覧ページにページャー UI が存在しない。ページャーを追加してすべての問題を閲覧できるようにする。

---

## 背景・目的

問題数が増えると一覧に全件が表示されず、後半の問題へアクセスできない。TICKET-009 で問題一覧ページを実装した際にページャー UI が含まれていなかったため、別チケットとして対応する。

---

## 受け入れ条件

- [ ] 問題一覧ページの下部にページャーが表示される
- [ ] 「前へ」「次へ」ボタン、および現在ページ・総ページ数が表示される
- [ ] 最初のページで「前へ」、最後のページで「次へ」がそれぞれ非活性になる
- [ ] タグフィルタ・キーワード検索と組み合わせてもページャーが正しく動作する
- [ ] `per_page` のデフォルト値は 20 とし、URL クエリパラメータ（`?page=2`）でページ状態を保持する
- [ ] ページ遷移時に画面上部にスクロールする

---

## サブチケット（コミット単位）

- [ ] `feat(frontend): ページャーコンポーネントを実装`
- [ ] `feat(frontend): 問題一覧にページャーを組み込み URL クエリパラメータで状態管理`
