## 概要

問題一覧APIにタグによるフィルタリングとキーワード検索クエリパラメータを追加する。ページネーションも合わせて実装し、大量の問題データを効率的に扱えるようにする。

---

## 背景・目的

問題数が増えると一覧取得だけでは目的の問題を探しにくくなる。タグフィルタリングとキーワード検索を組み合わせることで、学習したいテーマにすばやくアクセスできる体験を実現する。

---

## 受け入れ条件

- [x] `GET /api/v1/questions?tag_ids=1,2&keyword=暗号化` のようにクエリパラメータでフィルタリング・検索できる
- [x] 複数タグ指定時はAND絞り込みとする（OR検索が必要な場合は設計を明示）
- [x] キーワード検索は `title` / `body` / `explanation` / `memo` を対象とする
- [x] ページネーション（`page` / `per_page` クエリパラメータ）に対応し、レスポンスに総件数・現在ページ・総ページ数を含める
- [x] 検索結果が0件の場合は空配列と200を返す（404にしない）
- [x] `openapi.yaml` の `GET /api/v1/questions` にクエリパラメータの定義を追加済み
- [x] ユースケース層のユニットテストが作成されている
