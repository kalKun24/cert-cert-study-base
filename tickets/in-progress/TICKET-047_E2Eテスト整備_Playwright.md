## 概要

Playwright を導入し、チーム権限・招待・ルートガード等の主要フローを E2E テストで自動検証する。

---

## 背景・目的

TICKET-035〜038 で実装したチーム権限システムは手動テストで確認済みだが、
リグレッションを防ぐために自動化が必要。特にルートガード・権限分岐は
ユニットテストでカバーしにくいため E2E が有効。

---

## 受け入れ条件

- [ ] `frontend/e2e/` に Playwright テストが配置されている
- [ ] `npm run e2e` で全テストが実行できる
- [ ] 各テスト前後にテストデータを API 経由でセットアップ・クリーンアップする
- [ ] 以下のシナリオがすべてパスする（詳細はサブチケット参照）
