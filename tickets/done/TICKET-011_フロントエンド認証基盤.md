## 概要

ログイン画面・JWT 管理・認証済みルート保護・共通レイアウトを実装し、フロントエンド全機能の土台を整備する。このチケット完了後、各機能の UI チケット（TICKET-012〜014 等）が並行着手できる状態になる。

---

## 背景・目的

TICKET-002 で認証バックエンドは完成しているが、フロントエンドにはログイン画面すら存在しない。認証状態管理（JWT 保持・自動付与・有効期限切れハンドリング）がないと、他の画面開発ができない。

---

## 実装スコープ

### 1. ログイン画面
- username / password フォーム
- バリデーション（空欄・8 文字未満）
- 認証エラー時のメッセージ表示

### 2. JWT 管理
- ログイン成功時にトークンを `localStorage` に保存
- `axios` インターセプターで全リクエストに `Authorization: Bearer <token>` を自動付与
- 401 レスポンス時に自動ログアウト（トークン削除 → ログイン画面へリダイレクト）

### 3. 認証コンテキスト
- `AuthContext`（`user`, `token`, `login()`, `logout()`）を React Context で管理
- ページリロード時にトークンを復元し、認証状態を維持

### 4. 保護ルート
- 未認証状態で保護ルートにアクセスした場合、ログイン画面へリダイレクト
- ロール（admin / teamowner / user）に応じたルート制御

### 5. 共通レイアウト
- ナビゲーションバー（ユーザー名・ロール表示、ログアウトボタン）
- サイドメニューの骨格（各機能画面へのリンク）

---

## 受け入れ条件
